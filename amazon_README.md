# 📦 Amazon Delivery Delay Risk Prediction Dashboard

A production-style ML system that predicts delivery delay risk and provides
operational insights through an interactive Streamlit dashboard.

---

## 🗂️ Project Structure

```
amazon-delay-predictor/
├── data/               # Processed dashboard data (generated by train.py)
├── models/             # Saved model artefacts (generated by train.py)
├── app.py              # Streamlit dashboard
├── train.py            # Data pipeline + model training
├── predict.py          # Prediction module (used by app.py)
├── requirements.txt    # Python dependencies
└── README.md
```

---

## 🚀 Quick Start

### 1. Install dependencies

```bash
cd amazon-delay-predictor
pip install -r requirements.txt
```

### 2. Train the model

Place the raw Olist CSV files one directory up (i.e. in `data_project/`) then run:

```bash
python train.py
```

This will:
- Load and merge all 8 Olist datasets
- Engineer 30+ features (distance, temporal, synthetic operational)
- Train an XGBoost classifier
- Evaluate with classification report + 5-fold CV AUC
- Save `models/xgb_model.pkl`, `models/label_encoders.pkl`, `models/feature_names.pkl`
- Save `models/shap_mean_importance.csv`
- Save `data/dashboard_data.csv` for the dashboard

### 3. Launch the dashboard

```bash
python -m streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

## 🧠 Model Details

| Property | Value |
|---|---|
| Algorithm | XGBoost Classifier |
| Features | 30 engineered features |
| Training data | ~76K delivered orders |
| Class imbalance | Handled via `scale_pos_weight` |
| Evaluation | Classification report + ROC-AUC + 5-fold CV |

### Key Features
- `carrier_lag_hrs` – Time from approval to carrier pickup
- `approval_lag_hrs` – Time from purchase to approval
- `delivery_distance_km` – Haversine distance customer ↔ seller
- `driver_experience_yrs` – Synthetic, correlated with delay
- `prev_driver_delay_rate` – Historical driver reliability
- `estimated_days` – Delivery window tightness
- `traffic_level`, `weather` – Synthetic operational factors
