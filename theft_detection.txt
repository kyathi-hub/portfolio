📋 Project Summary: What is Happening?In a smart grid environment, Non-Technical Losses (NTL)—primarily caused by electricity theft, billing irregularities, and meter tampering—cause massive economic losses globally (estimated at $96 billion annually).  While the introduction of Advanced Metering Infrastructure (AMI) and smart meters provides detailed energy consumption data, manual methods of catching thieves (like video monitoring or physical inspections) are highly labor-intensive and impractical. Consequently, researchers are turning to Artificial Intelligence (AI) and Machine Learning (ML) to automatically mine smart meter data and isolate abnormal consumption patterns from benign ones.  However, a major bottleneck in this field is that real-world datasets containing actual theft cases are highly classified or unavailable for training and benchmarking these models.  🛠️ What We Are Doing in This ProjectTo solve this data scarcity problem and improve automated detection, this project executes the following core steps:1. Generating a Realistic Multi-Class Dataset (ETD2022)The Data Source: The project extracts authentic, real-world energy consumption data across 16 different consumer types (e.g., hospitals, restaurants, schools, warehouses) over a full year from the public Open Energy Data Initiative (OEDI) platform.  The Theft Generator: The authors modeled and coded a Python-based "theft generator" algorithm to inject six distinct types of mathematical electricity fraud behaviors into the clean dataset. These behaviors range from dropping consumption to zero during arbitrary periods to hour-dependent consumption reductions or time-shifting readings.  The Final Benchmark Set: The resulting dataset consists of 560,640 instances with 11 numerical features and one target class label, creating a major multi-class benchmark for the scientific community.  2. Developing and Evaluating the Detection SystemsThe researchers evaluated the dataset using four different validation protocols across five distinct Machine Learning techniques:$k$-Nearest Neighbors (KNN)  Decision Trees (DT)  Random Forest (RF)  Bagging Ensembles (BE)  Artificial Neural Networks (ANN)  The systems were tested in scenarios where the "Consumer Type" was explicitly known to the classifier, as well as an autonomous "Unknown Consumer" setup, proving the model can work robustly even if a building's type changes.  📊 Key Findings & OutcomesThe Best Model: The Random Forest (RF) model proved superior, outperforming the other contributing single models by 10% or more in performance metrics.  6-Class vs. 7-Class Tradeoff: The system performed dramatically better when executing a 6-class system (Normal + Theft 1 through 5), achieving a peak accuracy of 94.71%.  The "Theft 6" Bottleneck: The researchers discovered that Theft 6 (which reverses/swaps previous time readings) creates massive confusion with the "Normal" class because it closely mimics real, slightly shifted human behavior. Removing this class raised general system accuracy by 10%.
